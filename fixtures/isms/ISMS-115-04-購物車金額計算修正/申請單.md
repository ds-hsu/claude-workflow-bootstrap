### 變更需求 — 變更原因及內容說明

**問題現象**：
當會員於結帳頁輸入有效折扣碼（例 `SPRING100` 折 $100），且當前訂單商品金額已達免運門檻 $1,000 時，系統有約 3% 機率回傳錯誤的應付金額（比正確值多 $80）。此時使用者看到的金額包含運費，但金流請款扣款仍為正確金額，造成金額顯示與實際扣款不一致、客服接單量增加。

**重現步驟**：
1. 加入商品使小計 ≥ $1,000
2. 進結帳頁，選「宅配」
3. 輸入有效折扣碼
4. 在折扣金額套用的同時，系統偶發會保留運費 $80 於應付金額

**推測原因**：`CartService.CalculateTotal()` 方法在折扣碼 apply 後未重新執行免運判斷，存在 race condition。

### 資訊安全評估
- 金額計算錯誤雖未導致實際損失，但會降低會員信任
- 不涉及個資外洩、認證權限變更

### 變更程式碼摘要
```csharp
// 修正前
public decimal CalculateTotal(Cart cart, string promoCode) {
    var subtotal = cart.Items.Sum(i => i.UnitPrice * i.Quantity);
    var discount = _promo.Validate(promoCode, subtotal);
    var shipping = subtotal >= 1000 ? 0 : 80;
    return subtotal - discount + shipping;   // ← 折扣後可能跨過免運門檻卻未重算
}

// 修正後
public decimal CalculateTotal(Cart cart, string promoCode) {
    var subtotal = cart.Items.Sum(i => i.UnitPrice * i.Quantity);
    var discount = _promo.Validate(promoCode, subtotal);
    var afterDiscount = subtotal - discount;
    var shipping = afterDiscount >= 1000 ? 0 : 80;
    return afterDiscount + shipping;
}
```

### 測試計畫
1. 單元測試：新增 3 個 case 覆蓋「折扣後跨過門檻」「折扣後仍在門檻內」「無折扣碼」
2. 整合測試：執行完整結帳流程 E2E 腳本
